# Naming Rules

## Files And Directories

- Use kebab-case for file names and directory names.
- Keep generated build output under ignored build directories such as `dist/`.

## TypeScript Identifiers

- Use PascalCase for types, interfaces, classes, and enums.
- Use camelCase for functions, variables, parameters, and object property names unless an external API requires another shape.
- Avoid encoding file naming rules into TypeScript identifier names.
